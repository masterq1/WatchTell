Capabilities

Built for real-world deployment

Every feature is live in production — not a demo. The pipeline runs 24/7 on a single AWS account in us-east-1.

📷
Motion-triggered RTSP capture
OpenCV pixel-diff motion filter runs at configurable FPS. Only frames with real movement are uploaded, keeping S3 and SQS costs near zero between events.
MOTION_THRESHOLD · CAPTURE_FPS
🔤
OpenALPR on EC2 Spot
Full open-source ALPR stack — OpenALPR, OpenCV, Tesseract — compiled from source on Amazon Linux 2023 and baked into a reusable AMI. Max Spot price capped at $0.08/hr.
EC2 Spot · Custom AMI
🔄
Step Functions pipeline
Four Lambda stages in sequence: normalise the raw plate text, validate registration status via SearchQuarry with Upstash Redis caching, store to DynamoDB, check watchlist.
Step Functions Express · SQS
🔔
Watchlist alerts via SNS
Any plate on the watchlist triggers an SNS email within seconds of detection. Operator manages the watchlist through the dashboard or REST API with Cognito auth.
SNS · DynamoDB GSI
🖼️
Live event feed with thumbnails
The React dashboard displays every detected event with the captured keyframe thumbnail, plate tag, confidence score, timestamp, and camera ID in a real-time scrollable feed.
S3 Presigned URLs · CloudFront
📺
Live HLS video stream
go2rtc converts RTSP to HLS for browser delivery. HLS.js handles playback with automatic retry on session expiry — the feed recovers without user interaction.
go2rtc · HLS.js · auto-retry

Get started

Deploy in minutes

One CDK stack. Everything from S3 buckets to CloudFront to the EC2 Spot worker is provisioned automatically.

01 · BOOTSTRAP
# Clone and install dependencies
$ git clone https://github.com/masterq1/WatchTell
$ cd WatchTell
$ ./scripts/bootstrap.sh

# Installs Python deps, Node deps,
# and bootstraps CDK for your account
02 · DEPLOY
$ cd infrastructure
$ cdk deploy WatchtellSingle \
    --parameters CameraRtspUrl="rtsp://..." \
    --parameters CameraId="cam-driveway"

# First run builds OpenALPR AMI (~20 min)
# Subsequent deploys use the cached AMI
03 · FRONTEND (local dev)
$ cp frontend/.env.example frontend/.env.local
# Fill in VITE_API_URL, VITE_USER_POOL_ID,
# VITE_USER_POOL_CLIENT_ID from CDK outputs

$ cd frontend
$ npm install && npm run dev
04 · CAMERA RELAY
# Run on any device that can reach
# the camera and AWS
$ pip install opencv-python-headless boto3

$ CAMERA_ID=cam-driveway \
  RTSP_URL=rtsp://192.168.1.100/stream1 \
  MEDIA_BUCKET=watchtellsingle-media-xxx \
  QUEUE_URL=https://sqs... \
  python relay/camera_relay.py
